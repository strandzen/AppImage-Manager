# Settings Updates Walkthrough

## Changes Made
- **Phase 3 Forced Stabilization**: Resolved the final, persistent "log spam" in the terminal. I broke the `implicitHeight` recursion in `OverlaySheets` by strictly decoupling their content width from the parent container, and I eliminated the "graphics scene placement" warnings by delaying the initial page load until the window is fully initialized. The application terminal is finally quiet and clean.
- **Conditional List Separators**: Implemented a modern UI refinement where horizontal separators between list rows are automatically hidden when alternating row colors are enabled. This ensures a clean look without redundant visual lines.
- **List Clipping Fix**: Resolved issues where list rows and separators overlapped with the scrollbar. I added vertical separators for all scrollbars and adjusted delegate widths and paddings to ensure content stays within its intended boundaries.
- **Robust Model Connections**: Hardened the `TaskProgressPane` to handle cases where data models are not yet loaded, preventing `undefined` assignment warnings at startup.
- **Process List Spacing**: Increased the right margins for the process list headers and rows in the System Monitor. This provides much-needed breathing room between the "kill" icons and the vertical scrollbar, improving clickability and visual balance.
- **Debug Message Cleanup:** Removed the "🚀 MAINTAINER APP STARTING" debug print statement from `main.py` to keep the terminal output clean.
- **Global Font & Size Setting:** Added a new `Global Font` text input and a `Global Font Size` selection box to the "UI" settings category, with a note that 10 is standard/0 is default. These dynamically override `QGuiApplication`'s global font profile across the entire application instantly (backed by a realtime explicit binding on the `ApplicationWindow`). Leaving them empty/0 reverts to the default system font/size.
- **Package Removal Confirmation setting removed:** The "Confirm Package Removal" toggle has been fully removed from the QML interface (`qml/pages/SettingsPage.qml`) as well as the underlying Python manager (`models/settings_manager.py`).
- **UI Colors Label Renamed & UI Category Added:** Created a new "UI" section just before "UI Colors" and moved Corner Radius, Contrast Borders, and Alternating Row Colors out of General and into this new category.
- **Dynamic System Accent Color Fallbacks:**
  - Standardized all UI color choices (Emphasis, CPU, Memory, Download, Upload, Swap Bar) to explicitly fallback to `Kirigami.Theme.highlightColor` (the standard system accent color) when left completely empty. This fallback was applied in the `SettingsPage.qml` UI as well as across the entire app (`SystemMonitorPage.qml`, `TaskProgressPane.qml`, `CorpseCleanerPane.qml`).  - Made the `emphasisColor` setting **truly global**: Every descriptive text and label across the app that previously used `neutralTextColor` (which rendered as orange on some themes) or previously hardcoded colors now dynamically binds to `SettingsManager.emphasisColor` first. If left empty, they ALL fall back directly to the standard system accent color (`Kirigami.Theme.highlightColor`). This incorporates:    - The 'Cache / Trash' indicator in the storage visualizer.    - Descriptive sub-texts, package sources, and loading text in `AppImageManagerPane.qml`, `TaskProgressPane.qml`, `CorpseCleanerPane.qml` and `PackageManagerPane.qml`.    - System titles (Swap, OS, Host) in `SystemMonitorPage.qml`.    - The "Action Queue" header in `TaskProgressPane.qml`.  - Standardized all byte/size measurements across the app (AppImages, packages, tasks, corpses, and swap size) to strictly use the standard theme text color (`Kirigami.Theme.textColor`), keeping them separate from the emphasis color.  - Ensured the Swap Usage bar only follows the explicit `swapColor` setting (falling back directly to the system accent flavor), ignoring the `emphasisColor` setting entirely.
